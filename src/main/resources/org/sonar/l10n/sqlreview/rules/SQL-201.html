动态 SQL 拼接风险

使用 ${} 进行 SQL 拼接存在 SQL 注入风险，应使用 #{} 参数化查询。

风险说明

MyBatis 中的 ${} 会直接将参数值拼接到 SQL 中，攻击者可以构造恶意输入执行任意 SQL。

示例

<!-- 错误：SQL 注入风险 -->
<select id="getUser">
  SELECT * FROM app_user WHERE user_name = '${name}'
</select>

<!-- 正确：参数化查询 -->
<select id="getUser">
  SELECT * FROM app_user WHERE user_name = #{name}
</select>

建议

  • 始终使用 #{} 替代 ${}
  • 仅在动态表名、列名时使用 ${}，并做白名单校验
  • 使用预编译语句
