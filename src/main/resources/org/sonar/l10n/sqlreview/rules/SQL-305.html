Lambda 大表查询无 LIMIT

对超过阈值行数的大表执行 Lambda 查询，但没有 .last("LIMIT N") 或分页保护，可能返回海量数据。

风险说明

大表查询无 LIMIT 的问题：

  • 返回数据量过大，消耗大量内存和网络带宽
  • 可能导致 JVM OOM（内存溢出）
  • 数据库查询时间过长，影响服务响应
  • 在高并发场景下可能导致数据库连接池耗尽

示例

// 错误：大表查询无 LIMIT
Wrappers.<AppUserEntity>lambdaQuery()
    .eq(AppUserEntity::getStatus, 1)
    .list();  // app_user 有 15 万行，可能返回大量数据

// 正确：添加 LIMIT 保护
Wrappers.<AppUserEntity>lambdaQuery()
    .eq(AppUserEntity::getStatus, 1)
    .last("LIMIT 1000")
    .list();

// 正确：使用分页查询
Page<AppUserEntity> page = userService.page(
    new Page<>(1, 20),
    Wrappers.<AppUserEntity>lambdaQuery()
        .eq(AppUserEntity::getStatus, 1)
);

建议

  • 对大表查询始终添加 LIMIT 或使用分页
  • 使用 .last("LIMIT N") 或 MyBatis Plus 的 Page 分页
  • 对于批量处理，使用游标或分批查询
