Lambda WHERE 条件字段缺少索引

使用 MyBatis Plus Lambda Wrapper 的 .eq()、.like()、.in() 等方法构建的 WHERE 条件，其对应字段在生产数据库中缺少索引。

风险说明

条件字段缺少索引的问题：

  • 导致全表扫描，严重影响查询性能
  • 在大表上查询时间可能从毫秒级变为秒级
  • 增加数据库 CPU 和 IO 压力

示例

// 假设 app_user 表的 status 字段没有索引
Wrappers.<AppUserEntity>lambdaQuery()
    .eq(AppUserEntity::getStatus, 1)  // 如果 status 没有索引，将全表扫描
    .list();

// 修复：在 status 字段上创建索引
// CREATE INDEX idx_status ON app_user(status);

建议

  • 为常用的 WHERE 条件字段创建索引
  • 对于组合查询，考虑创建联合索引
  • 定期检查慢查询日志，发现缺少索引的字段
