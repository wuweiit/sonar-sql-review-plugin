大表查询无 LIMIT

对大表执行查询未设置 LIMIT，可能返回过多数据。

风险说明

没有 LIMIT 的查询可能返回数十万甚至数百万行数据，导致：

  • 内存溢出
  • 网络超时
  • 应用崩溃

示例

-- 错误：可能返回所有行
SELECT user_id, user_name FROM app_user WHERE status = 1;

-- 正确：限制返回行数
SELECT user_id, user_name FROM app_user WHERE status = 1 LIMIT 100;

建议

  • 所有查询都应添加 LIMIT
  • 使用分页查询处理大量数据
  • 设置合理的超时时间
