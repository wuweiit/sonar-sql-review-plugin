Lambda LIKE 以 % 开头

使用 MyBatis Plus Lambda Wrapper 的 .like() 方法时，查询值以 % 开头，导致无法使用索引。

风险说明

LIKE '%xxx' 模式的问题：

  • 无法利用 B-Tree 索引进行前缀匹配
  • 导致全表扫描，性能严重下降
  • 在大表上可能引发查询超时

示例

// 错误：前导通配符无法使用索引
Wrappers.<AppUserEntity>lambdaQuery()
    .like(AppUserEntity::getUserName, "%张")  // LIKE '%张' 全表扫描
    .list();

// 正确：右模糊可以使用索引
Wrappers.<AppUserEntity>lambdaQuery()
    .likeRight(AppUserEntity::getUserName, "张")  // LIKE '张%' 使用索引
    .list();

建议

  • 尽量使用右模糊（likeRight）代替全模糊（like）
  • 如果必须使用左模糊，考虑全文索引或搜索引擎
  • 对于前缀查询，使用 .likeRight() 或 .likeRight(Entity::getXxx, prefix)
